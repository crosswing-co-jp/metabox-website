# 🔍 cross-review — リリース前必須クロスレビュー会

ホームページの **新規ページ作成・既存ページの内容/レイアウト変更** を含む PR の **main マージ前に必ず実施** するクロスレビューです。

## 使い方

ユーザーから以下のいずれかが渡されます:
- **URL** (例: `http://localhost:3000/pr/ivent_shien/`、`https://crosswing-co-jp.github.io/hope21jp-website/pr/ivent_shien/`)
- **ファイルパス** (例: `pr/ivent_shien/index.html`)

URL / パスが提供されていない場合は、ユーザーに「対象ページの URL またはパスを教えてください」と聞きます。

ローカル http.server (3000) や Preview (GitHub Pages) を立ち上げてからレビュー実行することを推奨します。

**ページ単位で評価**します (diff だけでなく、ページ全体を通読)。

---

## 6 視点レビュアー (並列召喚)

`Agent` tool を **同時に 6 つ呼び出す** (1 つの message に 6 つの tool call) ことで並列実行します。

各 Agent には以下の役を演じさせ、与えられた評価軸で 300-500 words のレビューを書かせます。

### 1. 同人誌作家 (一次読者)

**ロール**: 同人誌即売会に参加するサークル参加者・主催者として、初めてこのページを訪れた人。前提知識ゼロ。

**評価軸**:
- 3 秒テスト: ファーストビューだけで「何のページ・自分得」が分かる?
- 「うち対象?」即答性: サークル参加者・主催者の両方で 30 秒以内に結論出せる?
- メリット理解: ベネフィット数値・例示が刺さる?
- 申込みフロー: 動線で迷子にならない?
- 読みやすさ: 言葉の温度感、堅すぎ・冷たい箇所
- 不安・疑問・離脱ポイント

### 2. Web ライター (10 年経験編集者)

**ロール**: Web コピーライター・出版編集経験 10 年。校正のプロ。

**評価軸**:
- リード文の引き: ファーストビューが読者を引き込むか
- 見出し対称性: h2/h3 が機械的すぎないか、AI 的整然さがないか
- 冗長・重複: 同じこと言ってる箇所、削れる文
- NG ワード: 「いただけます」「ご活用ください」「以下の」「下記」「ぜひ」「重要なのは」など
- トーン: 業務サイトとして適切な敬語密度、対象読者別の使い分け
- 箇条書き過剰: ul/li を頼りすぎてないか、文章で語るべき箇所
- 数字・固有名詞の具体性

### 3. Web UI デザイナー (UX/UI ベテラン)

**ロール**: 10 年以上の UX/UI 設計経験。コンバージョン LP の設計に強い。

**評価軸**:
- ファーストビュー完成度: KV、CTA、目線の動き
- ビジュアルヒエラルキー: h2/h3/p の重み、目線誘導
- CTA: 本命 CTA と目次系の差別化、配置と頻度、押下動機
- 余白・密度: 詰まりすぎ・スカスカ
- モバイル想定: 各セクションのモバイル表示推察
- アクセシビリティ: alt属性、コントラスト比、見出し階層
- 競合他社サイトとの比較 (必要なら WebFetch で参照)

### 4. グラフィックデザイナー (ビジュアル)

**ロール**: 印刷・Web 両方の経験あるグラフィックデザイナー。ブランド設計のプロ。

**評価軸**:
- ブランドカラー一貫性: 各色の役割分担 (ピンク/橙/黄/赤)、過剰使用
- 装飾過剰・装飾不足: ボックス・アイコン・絵文字の濫用
- 図像: 自社写真 vs ストック画像、品質、必要性
- AI デザインクセ (兆候 20 個):
  - 中央寄せ Hero + CTA 2 個 / 3 カラム特徴 / 鉄板セクション順
  - グラデーション Hero / ガラスモーフィズム / Tailwind 既定色 / rounded-2xl 連発
  - Inter / Geist 統一フォント / Lucide アイコン統一
  - hover:scale-105 / scroll fade-in / 「○○を、もっと簡単に。」構文

### 5. 校正者 (出版編集 10 年)

**ロール**: 出版業界 10 年の校正者。表記揺れ・敬語密度・固有名詞統一にうるさい。

**評価軸**:
- NG ワード grep カウント (「いただけます」「ご確認ください」「ご活用」「以下の」「下記」「いたします」「ぜひ」「重要なのは」)
- 業界用語の使い方: オンリーイベント / プチオンリー / 親イベント / 直接搬入 / スペースNo. 等
- 表記揺れ: 「1ヶ所/1か所/1箇所」「サークル様/さん」「弊社/当社/ホープツーワン」等
- 固有名詞統一: 製品名・サービス名・主催名
- 敬語密度: 過剰敬語 vs カジュアル化のバランス
- 「※」注釈の数と配置 (連続 3 個以上は読み飛ばされる)

### 6. AI クセ検出官

**ロール**: AI 生成物特有のクセを検出する専門官。

**判定基準** (引用):
- `hope21-secretariat/standards/ai-output-quality.md` の **18 兆候**
  - 構造: 見出し綺麗すぎ / 冒頭で全体地図 / 段落長さ揃いすぎ / まとめで締める / 箇条書きとテーブルで埋まる
  - 文体: 「重要なのは」連発 / 「以下のように」多用 / 賞賛副詞 / 「ぜひ」「ご活用」 / 「〜することができます」 / 「皆様におかれましては」 / 問いかけ定型
  - 内容: 両論併記 / 個人体験が抽象的 / 数字に幅 / 感情が抜けてる / 時間軸が「先日」 / 完璧で間違いなし
- `hope21-secretariat/standards/ai-design-quality.md` の **20 兆候**

**出力**:
```
## AI 自己レビュー
- NG ワードヒット: 「○○」(N 箇所)
- 18 兆候該当: #X (○○), #Y (○○)
- 20 兆候該当: #X (○○)
- 推奨修正: ○○ を ○○ に
- AI 認定度: N/10 (10 = AI 製と即バレ、1 = 完全人間製レベル)
```

## 各 Agent に渡すプロンプト雛形

```
あなたは [ロール] です。hope21.jp の [ページ名] ページをレビューします。

レビュー対象: [URL or path]

取得方法:
- URL の場合: `curl -s [URL] > /tmp/cross-review-page.html` で取得
- パスの場合: Read tool で読み込み

[評価軸を箇条書きで列挙]

300-500 words で、辛口で OK、迎合不要。

**出力形式**:
1. ★ X.X / 5.0 (リリース判定)
2. Tier S (リリースブロッカー): ...
3. Tier A (リリース前推奨修正): ...
4. Tier B (リリース後でも OK): ...
5. 一言コメント
```

## リリース判定基準

- **★ 4.5 以上 (全員 ★ 4.0+)** → リリース GO ✓
- **★ 3.0 以下が 1 名でも** → Tier S あり、修正必須 ✗
- **AI 認定度 7/10 以上** → Tier A 修正推奨
- **平均 ★ 4.5 以上** → 高品質 ✓✓
